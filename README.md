# RTL Problems & Solutions

Welcome to my RTL Problems & Solutions repository.

## 📖 About

This repository is my personal collection of RTL (Register Transfer Level) design problems that I encounter while learning, practicing, and working on digital design.

For every interesting RTL problem, I aim to document:

- The original problem statement
- My understanding of the problem
- My thought process before writing the RTL
- The final RTL implementation
- The corresponding testbench
- Key learnings and observations

The goal is not just to solve problems, but to build a knowledge base that captures **how** I approached each problem.

---

## 🎯 Objectives

- Strengthen RTL design skills
- Practice Verilog/SystemVerilog coding
- Improve problem-solving techniques
- Record design decisions and trade-offs
- Build a reusable reference for future learning
- Share solutions that may help other RTL engineers

---

## 📂 Repository Structure

Each problem follows the same directory structure:

```text
Problem_Name/
│
├── design/
│   └── RTL source files
│
├── testbench/
│   └── Testbench and simulation files
│
└── README.md
    ├── Problem statement
    ├── Design requirements
    ├── My thought process
    ├── Solution approach
    ├── Assumptions
    ├── Edge cases
    └── Key learnings
```

---

## 📝 What Each Problem Contains

### 📁 design/

Contains the RTL implementation of the solution.

### 📁 testbench/

Contains the verification environment used to validate the design.

### 📄 README.md

Documents the complete journey of solving the problem, including:

- Problem description
- Requirements
- Initial ideas
- Design decisions
- Challenges faced
- Alternative approaches (if any)
- Final solution explanation
- Lessons learned

## 🚀 Purpose
